Introduction to Rust

A systems programming language that runs blazingly fast

Welcome everyone! Today we'll explore why Rust is gaining so much traction in systems programming.

Why Rust?

  • Memory safety without garbage collection
  • Zero-cost abstractions
  • Fearless concurrency
  • Rich type system and ownership model
These are the four pillars of Rust. Click through each one.

Rust vs C++

Rust

  • Ownership system
  • No null pointers
  • Pattern matching
  • Cargo package manager

C++

  • Manual memory management
  • Null pointer exceptions
  • Switch statements
  • CMake / Conan / vcpkg

fn main() {

}

// With error handling
fn read_file(path: &str) -> Result<String, std::io::Error> {
    std::fs::read_to_string(path)
}
Notice how Rust uses Result types for error handling instead of exceptions.
